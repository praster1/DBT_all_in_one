# 🎨 DBT All In One 스타일 시스템 v1

> 문서 전체의 시각/구조 일관성을 유지하기 위해, **수동 서식이 아닌 Named Style 중심**으로 편집합니다.

---

## 1) 목적

- 장/부록/레퍼런스 간 톤과 가독성 일관성 확보
- 작성자 변경 시에도 산출물 품질 편차 최소화
- PDF/DOCX 렌더링 품질 안정화

## 2) 필수 스타일 세트

| 그룹 | 스타일 |
|---|---|
| 제목 계층 | `Title`, `Subtitle`, `Part Title`, `Chapter Title`, `Heading 1~3` |
| 본문 계층 | `Body`, `Body Small`, `Bullet List`, `Numbered List` |
| 코드/명령 | `Code Block`, `Command Block` |
| 표/보조 | `Table Header`, `Table Body`, `Caption` |
| 콜아웃 | `Note`, `Warning`, `Example`, `Quote` |

## 3) 본문 규칙

1. 본문 정렬은 **좌측 정렬**(문서 전체 고정)
2. 첫 줄 들여쓰기는 **전역 한 가지 방식만** 사용
3. 목록은 **hanging indent** 고정
4. 표는 셀 패딩/헤더 강조 방식 고정

## 4) 코드/명령어 규칙

1. 항상 좌측 정렬
2. 모노스페이스 폰트 고정
3. 배경색/테두리/상하 여백 통일
4. 언어 라벨 표기 통일: `sql`, `bash`, `yaml`, `json`, `python`, `text`
5. 실행용 블록에는 **파일 경로 + 실행 순서 + 기대 결과**를 함께 명시
6. 긴 명령어는 줄바꿈(continuation) 방식 통일

## 5) 폰트/컬러 최소 원칙

- 폰트는 최대 3종: 본문용 1 / 제목용 1 / 코드용 1
- 색상 팔레트 최소화
  - 기본 텍스트
  - 제목/강조
  - 코드/박스 중립
  - Warning/Note 포인트

## 6) 콜아웃 정책

| 유형 | 용도 |
|---|---|
| Note | 판단 팁, 운영 권장사항 |
| Warning | 장애 위험, 호환성 리스크 |
| Example | 예제 트랙 연결 설명 |
| Quote | 정의/원칙 인용 |

> 동일한 정보 유형에는 동일 콜아웃만 사용합니다. 임의 박스 스타일 추가는 금지합니다.

## 7) 금지 규칙

- 수동 폰트 변경(부분 강조 제외)
- 임의 줄간격/문단 간격 남발
- 코드 블록 중앙 정렬
- 장마다 다른 표 스타일 사용
- `이 장의 로드맵` 박스의 기계적 반복

## 8) 스타일 변경 절차

1. 스타일 이슈 발견
2. 개별 문단 직접 수정 대신 스타일 정의부터 수정
3. 샘플 2개 챕터에 시범 적용
4. 전체 문서 일괄 반영
5. PDF/DOCX 렌더 검수
